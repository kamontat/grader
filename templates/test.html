Getting started

Start working on problems by selecting one on the left. Number in the parenthesis indicates point received from completing that problem.

You can start working on any problem you want, not just in the order they're presented.

Submission

There are 2 types of submissions: grader and Codejam-style submission. Some problem or test set may not allow you to use one of them.

Grader submission

In grader submission, you have to submit your source code that retrieve input from standard input and send output to standard output. Standard error is ignored so you may leave your debug messages.

You may submit source code in Python 2.7, PHP 5.4, Java SE7, JavaScript (NodeJS 0.10). Exact version of the runtime may vary and some languages may not be available in some test set or problem.

After the source code was submitted, your code will be put into a queue. When it is your turn the system will compile and/or run your code, perhaps multiple times automatically and report the result. You don't have to refresh this page.

Codejam-style submission

In this type of submission, you will be given a input file which you have to produce output from it following the problem specification.

In some problem the system will give you multiple test cases at once. The first line of such test case will be a number N and N test cases will follow. This may not be indicated in the problem statement as grader submission system automatically addresses for this.

You will have a time limit for producing the output. If the time limit is up you can ask for more retries which you will be given a different test case.

Some problem may only allow you to use codejam-style submission for practice and will not give you credits for the problem. This will be indicated in the problem in red box.

The output from codejam-style submission are the same as grader submission.
